Project: Common Ground — Icebreaker Game Platform
Stack: React + TypeScript + Vite (Vercel) / FastAPI + MongoDB Atlas (Render)
Repo: https://github.com/Danino42/common-ground
Frontend: https://common-ground-three.vercel.app
Backend: https://common-ground-backend-q6e1.onrender.com

Current Architecture
Auth: Email OTP only (no password). First login auto-creates account. JWT in localStorage (7 days). SendGrid HTTP API for OTP emails.
Card sets: Stored in MongoDB card_sets collection. Games embed cards directly at start time. Players never fetch sets separately.
Deck hash: base64 of {n: name, c: [card texts]} — stored in MongoDB, shown truncated in preview modal with copy button.
Game modes: Swipe, Circle (offline), Random Groups, Topic by Topic.

Backend Files
app/routers/facilitators.py

POST /facilitators/login/request — upserts facilitator, sends OTP via SendGrid HTTP API
POST /facilitators/login/verify — verifies OTP, returns JWT
GET /facilitators/me — returns email + saved_card_sets
GET /facilitators/me/saved-sets — returns saved set IDs
PATCH /facilitators/me/saved-sets — updates saved set IDs

app/routers/card_sets.py
Full CRUD + save/unsave:

GET /card-sets/ — lists sets visible to user (system + public + own), includes saved boolean
GET /card-sets/{id} — get by MongoDB ID
POST /card-sets/ — create, generates deck_hash, auto-saves to facilitator
PATCH /card-sets/{id} — update (own sets only), regenerates hash
DELETE /card-sets/{id} — delete (own sets only)
POST /card-sets/{id}/save — add to saved
DELETE /card-sets/{id}/save — remove from saved

app/routers/games.py

start_game fetches card set from MongoDB, embeds cards array directly in game document
get_results returns cards field so frontend never needs to look up sets separately

app/seed.py
Seeds 6 premade sets into MongoDB. Run once:
bashcd backend
python -m app.seed
app/database.py
Exports db (AsyncIOMotorClient) and games = db["games"]
